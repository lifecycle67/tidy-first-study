* 설명하는 변수
  * 복합한 표현식을 의미 있는 이름의 변수로 분해 하는 것.
* 설명하는 상수
  * 의미 없는 숫자/문자 리터럴을 이름 붙인 상수로 변경
    * 예제
      * 최대 로그인 시도 횟수
      * HTTP 상태 코드 명
* 명시적인 매개변수
  * 함수가 필요한 인자는 명시적으로 전달받자
    * 전역변수, ThreadLocal, currentUser 같은 것을 몰래 쓰는 함수
* 비슷한 코드끼리
  * 긴 코드 덩어리에서 의미가 비슷한 코드끼리 배치 하는 방법
* 도우미 추출
  * 목적이 분명하고, 다른 코드와 상호 작용이 적은 코드 블록을 작은 함수로 분리하는 방법
* 하나의 더미
  * 여러 조각으로 흩어져있고, 이해하기 어려운 코드는 다시 필요한 만큼의 더미로 모았다가, 어려운 부분을 추출하여 새롭게 정리하고자 하는 방향
    * 길고 반복되는 인자 목록
    * 반복되는 코드, 그 중에서도 반복되는 조건문
    * 도우미에 대한 부적절한 이름
    * 공유되어 변경에 노출된 데이터 구조
* 설명하는 주석
  * 코드만으로는 의도/이유가 잘 안보인다고 판단될때 주석을 활용
  * 다른 사람의 관점에서 이해할 수 있도록
* 불필요한 주석 지우기
  * 코드만으로 내용을 이해할 수 있다면 주석은 삭제한다.
  * 코드 변경으로 인해 더이상 필요없거나 중복되는 주석을 삭제한다.
