# Chapter 21: 코드 정리 시점

## 정리하지 않기
앞으로 변경 할 일이 없다면. 지금 돌아가는것 잘 돌아가서 그대로 쓰면 되고, 추가적으로 변경이 필요한 요구사항이 딱히 없어서 수정할 일이 없다면, 정리를 하지 않아도 되겠다 라는 생각.

## 정리 미루기
코드 정리가 필요할 정도로 뒤죽박죽인 것은 있는데, 일단 요구사항 먼저 마감 전 까지 다 처리해야 할 때. 그냥 미루는 것이 아니라, 체크리스트 같은것을 하나 만들어서 정리 해 두고, 나중에 시간이 날 때 하나씩 꺼내다가 정리하는 것인데, 이것은 업무 하면서 실천 해 보면 괜찮을 것 같다는 생각이 들었다. 

## 동작 변경 후에 코드 정리
동작 변경과 코드 정리를 꼭 나눠서 해야 하나 하는 생각이 든다. 동작 변경 하면서, 어떻게 하면 좀 더 이해하기 괜찮게 할 수 있을 지 고민해서 코드 정리도 같이 하면 된다고 생각한다. 물론 시간에 정말 쫓기면 동작 변경만 먼제 해야 하는 경우도 있다. 로그 찍는것 여기저기 끼워넣는 경우도 있고, 넘겨주는 인자만 바꿔주는 경우도 있곤 할 것 같다. 그럴 때는 나중에 할 코드 정리 체크리스트에 넣어두면 되겠다 하는 생각이 들었다.
