# Chapter 1: 보호구문
조건문, 특히 중첩되면서도 함수나 메소드 등의 시작 부분에 있어서 먼저 검사를 하는 용도로 들어가 있는 조건문이 있다면, 조건을 만족하지 않으면 리턴을 하는 형태로 넣어 보호구문 형태로 작성하는 것.
개인적으로 보호 구문을 API 컨트롤러 작성 하면서 유효성 검사 용도로 많이 사용한 기억이 있는데, depth 가 1 정도로 항상 유지 되어서 읽기 편하긴 했던 것 같다. 다만 여러 예외 케이스 처리 하자고 생각보다 많이 넣게 되면 읽이 어려워지는 건 마찬가지인 것 같다. 책에서는 도우미 함수로 추출 해 보라는 내용도 있는데, 여기저기 많이 쓰이는 보호 구문이라면 그렇게 해 보는것도 좋을 것 같다.
