# Chapter 22: 요소들을 유익하게 관계맺는 일

소프트웨어의 구성요소에 대해 설명하고, 이들을 관계맺는 방식. 그리고 어떻게 유익한 방향으로 관계맺을 수 있는지에 대한 내용이였다. 알고는 있고, 들어본 적은 있었지만 딱히 깊게 생각 해 보진 않은 요소 단위에 대해 알아보고.

토큰: var, func, return 같은 키워드
식: `maxNum.isNull`, `maxNum + 4` 와 같은 토큰 조합으로 값 하나가 만들어 지는 것
문: `maxNum = 10`, `for(...){...}`, `if...` 와 같이 실제 작업을 수행하는 토큰 조합.
함수: 식, 문을 모아둔 블록으로 호풀하여 재사용 가능한 작업(일반적으로 입출력 있음)
모듈과 객체: 모듈은 파일 하나에 관련된 함수와 변수 등을 모아둔 것. 객체는 데이터와 동작을 묶어둔 것.
시스템: 위 요소를 다 모아둔 것으로 완전한 하나의 앱이나 서비스 등을 구성하는 집합.

이들 사이 관계를 만드는 방법도 알아보았다. 호출, 참조, 발행, 대기 같은 생각 해 보면 또 익숙한 개념이였다.

중요한건 그래서 관게를 어떻게 맺을 것인가에 대한 내용인 것 같다. 책에서는 간단히 어떤 함수에서 다른 클래스에 있는 두 메소드를 호출해서 쓰는것을, 해당 클래스 안에 별도 메소드를 하나 더 만들어 옮기고, 이를 기존 함수에서 대신 호출하는 형태로 관계를 조정하는 예시를 보여 주었다.

어떤 클래스나 모듈에서 여러 함수를 호출해다가 사용하는데, 이를 동일한 패턴으로 여러곳에서 쓰는 경우가 있다면 책에서 처럼 별도의 메소드로 분리할 수도 있고 (도우미 함수 느낌). 그 외에도 다양한 방법으로 관계를 조금씩 조정하면서 더 유익한 방향으로(?) 하면 되겠다는 생각이 들었다.
