# Chapter 10: 명시적인 매개변수
처음에 예제를 보고 이것이 왜 문제가 될까 하는 생각을 잠깐 했다. 자세히 보니 딕셔너리나 맵 형태의 자료를 매개변수 용도로 써서 넘기는 것이였는데, 그러면 사실 매개변수에 무엇이든 들어갈 수 있으니, 문서화가 잘 되어있지 않다면 함수 시그니처만 보고 어떻게 호출해야 하나 알기 어렵게 되는 문제가 생길 수 있다는 생각이 들었다.

코드를 작성하다 보면, 뭐랄까 함수 하나에 매개변수 생각보다 많은 수를 받는 경우가 종종 있는 것 같다. 그럴 때, 맵이나 딕셔너리 보다는, 매개변수 담을 클래스나 레코드 타입 하나 정의해서 객체 형태로 받으면 좀 간결해 지면서도 호출 어떻게 해야 하는지도 명확해 질 것 같다는 생각이 들었다.
