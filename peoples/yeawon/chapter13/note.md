## 13장. 하나의 더미

- 코드가 조각조각 흩어져 있으면 코드를 전체적으로 이해하기 어렵다. → **필요한 만큼의 코드를 하나의 더미(One Pile)처럼 느껴질 때까지 흩어진 코드를 모아라. 그리고 나서 깔끔하게 정리하라**
- 이론적으로 말하자면, 코드 정리를 선행하면 더 작은 단위로 결합을 제거하는 길을 제시하여 응집도를 높일 수 있다
- 실무적으로 말하자면, 한 번에 머릿속에 기억하고 있어야 할 코드의 상세 내용을 줄여준다
- 작은 코드 조각을 지향하는 목적은 코드를 한 번에 조금씩 이해할 수 있도록 하지만, 때때로 이 과정이 잘못될 수도 있다.
- 작은 코드 조각들이 서로 교류하는 방식은 코드를 더 알기 어렵게 합니다. 명확성을 되찾으려면, 먼저 코드를 한데 모아서 이해하기 어려운 부분은 추출해서 새롭게 정리해야 한다
- **✅ 다음 증상들을 찾아보자**
  - 길고 반복되는 인자 목록
  - 반복되는 코드, 그 중에서도 반복되는 조건문
  - 도우미에 대한 부적절한 이름
  - 공유되어 변경에 노출된 데이터 구조
- **코드가 이해하기 어려울 때, 작은 코드 조각들을 한데 모아라.** (자동화된 리팩토링 도구 사용 또는 수작업) → 코드를 먼저 정리할까? 눈에 보이는 것만 먼저 변경할까?
