## 8장. 설명하는 변수

1. 어렵고 크고 복잡한 코드의 표현식 -> **전체에서 일부 표현식을 추출**한 후, **표현식의 의도가 드러나도록 변수 이름을 할당해 보자**
    
    ex)
    ```python
    return new Point(
        ...긴 표현식...,
        ...다른 긴 표현식...
    )
    ```
    ⇒ **일부 표현식 추출**하여 **설명하는 변수로 바꾸기**
    ```python
    x := ...긴 표현식...
    y := ...다른 긴 표현식...
    return new Point(x, y)
    ```
2. 또는 **표현식 의미를 더 구체화**
    - ex) 너비와 높이, 위쪽과 된쪽, 뛰기와 오르기 등의 변수명으로 쓰는 것

- 설명하는 변수를 적용하면, 이제 표현식과 분리되었기 때문에 다음 코드 변경 시, 둘 중 하나만 읽으면 됨. ∴ 보다 빨리 변경 가능
- 언제나 그렇듯, <u>코드 정리에 대한 커밋</u>과 <u>동작 변경에 대한 커밋</u>은 **분리**하라
