## 4장. 새로운 인터페이스로 기존 루틴 부르기

- 루틴을 호출해야 하는데, 기존 인터페이스 때문에 어려움이 있다면 → **호출하고 싶은 인터페이스를 새롭게 구현하여 호출하라**
- 통로 인터페이스(pass-through interface)
    - 새로 만든 인터페이스는 그저 기존 인터페이스를 호출하는 것으로 구현할 수 있음
    - 기존 인터페이스 호출 코드 → 모두 새로운 인터페이스 호출 코드로 이전 후, 기존 인터페이스 코드 제거 → 새 인터페이스가 직접 루틴을 구현하도록 변경할 수 있음
- **거꾸로 코딩하기**: 루틴의 마지막 줄부터 시작해보라. 마지막 줄에 이르기까지 필요한 결과는 모두 확보한 듯이 짜는 것
- **테스트 우선 코딩**: 테스트부터 작성하여 통과 요건 정하기
- **도우미 설계**: 만일 나에게 특정 업무를 해주는 루틴, 객체, 서비스가 있다면 나머지 작업은 쉬워짐
