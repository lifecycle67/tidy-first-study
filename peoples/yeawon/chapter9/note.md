## 9장. 설명하는 상수

- **상징적인 상수를 만들어라** 
- <u>리터럴 상수(Literal constant)로 사용된 곳</u> -> **상징적인 상수(Symbolic const)로 바꿔라**
    - *<u>리터럴 상수</u> or <u>리터럴</u>: 소스 코드에 기록된 값의 텍스트 표현*
    - *<u>상징적인 상수</u>: 값이 바뀌지 않는 제약이 있지만, 변수처럼 고정 값 클래스 중 하나를 취할 수 있는 기호(symbol)를 써서 상수를 정의한 것*
- Good example

    ```python
    # As-is
    if response.code = 404  # 404: 리터럴 상수
        ...코드...

    # To-be
    PAGE_NOT_FOUND := 404   # 상징적인 상수. 리터럴에 이름을 붙인 것
    if response.code = PAGE_NOT_FOUND
        ...코드...
    ```
- Bad example

    ```python
    ONE = 1
    ...ONE... # 하나가 필요할 때 어디든 등장
    ```
    > 같은 리터럴 상수는 같은 의미로 쓰여야 한다.

- <u>한번에 바뀌어야 하거나 함께 이해해야 하는 상수들</u>을 **한곳에 모아두고**, <u>다른 이유로 묶인 변수들</u>을 **분리하는 작업**을 진행한다
- 결합도, 응집도는 일단 뒤로 하고 작업해라
